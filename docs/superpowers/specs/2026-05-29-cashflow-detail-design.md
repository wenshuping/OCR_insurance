# 个人现金流明细表 — 设计文档

> 日期: 2026-05-29
> 状态: 已确认
> 方案: 纯前端计算（方案 A）

## 概述

为 OCR 保险系统新增"个人现金流明细"页面，展示被保人的确定性现金流年度明细和场景赔付明细。支持两层视图（单保单明细 + 被保人年度汇总），纳入现有报告导出（PDF/长图）。

### 需求决策

| 维度 | 决定 |
|------|------|
| 范围 | 两层：单保单明细 + 被保人年度汇总 |
| 页面 | 新建独立页面 |
| 表格类型 | 年度现金流表 + 场景赔付表 |
| 数据 | 先修指标数据，再开发 UI |
| 入口 | 从家庭保障总览页面进入 |
| 导出 | 纳入现有报告 PDF/长图导出 |

## 1. 数据模型

新增 TypeScript 类型，定义在 `src/api.ts`。

### CashflowEntry — 年度现金流条目

```typescript
type CashflowEntry = {
  year: number;              // 日历年份，如 2030
  age: number;               // 当年年龄
  amount: number;            // 当年领取金额
  cumulative: number;        // 累计领取
  liability: string;         // "生存保险金" | "养老年金" | "满期生存保险金"
  policyId: number;          // 来源保单 ID
  productName: string;       // 来源产品名称
  calculationText: string;   // 计算公式，如 "基本保额 = 1,465元"
};
```

### ScenarioEntry — 场景赔付条目

```typescript
type ScenarioEntry = {
  scenario: string;          // "一般意外身故/全残"、"护理金，18岁前"
  formula: string;           // "60,000 × 10"
  amount: number;            // 600,000
  condition: string;         // 附加条件
  policyId: number;
  productName: string;
  calculationText: string;   // "基本保额 × 10 = 600,000元"
};
```

### PolicyCashflowPlan — 单保单现金流计划

```typescript
type PolicyCashflowPlan = {
  policyId: number;
  productName: string;
  company: string;
  insured: string;
  insuredBirthday: string;
  effectiveDate: string;
  annualEntries: CashflowEntry[];    // 年度明细（可为空）
  scenarioEntries: ScenarioEntry[];  // 场景明细（可为空）
  totalDeterministicCashflow: number;
  expired: boolean;                  // 是否已过期
};
```

### MemberAnnualSummary — 被保人年度汇总

```typescript
type MemberAnnualSummary = {
  member: string;
  birthday: string;
  entries: MemberYearEntry[];
  totalCashflow: number;
};

type MemberYearEntry = {
  year: number;
  age: number;
  totalAmount: number;       // 该年所有保单领取合计
  cumulative: number;        // 累计领取
  details: CashflowEntry[];  // 当年各保单明细
};
```

### 设计要点

- `CashflowEntry` 只在有确定领取金额时才生成条目，现金价值/账户价值缺失时不硬写
- `PolicyCashflowPlan` 一个保单可以同时有 `annualEntries` 和 `scenarioEntries`
- `MemberAnnualSummary` 汇总同一被保人所有保单的年度现金流

## 2. 核心计算引擎

全部纯函数，放在 `src/App.tsx`，与现有 `buildFamilyCoverageOverview()` 风格一致。

### 2.1 主函数 buildPolicyCashflowPlans()

```
输入: Policy[]
输出: PolicyCashflowPlan[]
```

流程：

```
对每张保单：
  ├─ 提取基础参数（生日、生效日、缴费年期、基本保额、年交保费）
  ├─ 判断是否过期（coveragePeriod 结束日期 < 当前日期）
  ├─ 遍历 coverageIndicators
  │   ├─ 现金流类指标 → expandCashflowIndicator() → CashflowEntry[]
  │   ├─ 意外/疾病/护理类指标 → buildScenarioEntry() → ScenarioEntry[]
  │   └─ 跳过：现金价值/账户价值缺失的、非给付型指标
  ├─ 按年份排序 annualEntries，计算 cumulative
  └─ 汇总 totalDeterministicCashflow
```

### 2.2 年份展开 expandCashflowIndicator()

```
输入: CoverageIndicator + Policy 参数
输出: CashflowEntry[]
```

解析 condition 文本确定领取年份范围：

| condition 文本模式 | 解析规则 | 示例 |
|---|---|---|
| `生效满N年` | startYear = effectiveYear + N | 生效满5年 → 2030 |
| `N周岁后首个保单周年日` | startYear = birthYear + N（取整到下一个周年日） | 55周岁后 → 2043 |
| `保障至YYYY-MM-DD` / `届满前` | endYear 从 coveragePeriod 解析 | — |
| `每年` / `每年给付` | frequency = 1，按年展开 | — |

#### 盛世恒盈年金计算示例

```
生存保险金：
  condition → "生效满5年首个周年日到养老年金开始前"
  startYear = 2025 + 5 = 2030, endYear = 2042
  amount = resolveIndicatorAmount(indicator, policy) = 1,465
  → 生成 13 条 CashflowEntry (2030-2042)

养老年金：
  condition → "女性55周岁后首个保单周年日到届满前"
  startYear = 1988 + 55 = 2043, endYear = 2072
  amount = 1,465
  → 生成 30 条 CashflowEntry (2043-2072)

满期生存保险金：
  condition → "保障期满"
  year = 2073
  amount = 11,000 × 10 = 110,000
  → 生成 1 条 CashflowEntry (2073)

确定现金流合计 = 19,045 + 43,950 + 110,000 = 172,995
```

#### 计算口径

- 实际交纳保险费：按对应险种算，不直接整单乱乘
- 主险年交：`年交保费 × 缴费年期`
- 趸交附加险/万能账户：只算 1 次
- 现金价值、账户价值：没有入库数值时不展示确定金额
- 现金流：只展示可确定的生存金、养老年金、满期金

### 2.3 场景构建 buildScenarioEntries()

对非现金流类指标（意外、疾病、护理），逐条生成 ScenarioEntry。

#### 畅行万里智赢版

```
满期生存保险金     → "3,156 × 10"       = 31,560
疾病身故/全残 41岁前 → "31,560 × 1.6"   = 50,496
疾病身故/全残 41-61岁 → "31,560 × 1.4"  = 44,184
疾病身故/全残 61岁后 → "31,560 × 1.2"   = 37,872
一般意外身故/全残   → "60,000 × 10"    = 600,000
步行/骑行交通意外   → "60,000 × 15"    = 900,000
驾乘意外           → "60,000 × 20"    = 1,200,000
高空坠物/抛物意外   → "60,000 × 20"    = 1,200,000
客运轮船/汽车意外   → "60,000 × 30"    = 1,800,000
电梯意外           → "60,000 × 30"    = 1,800,000
公共场所特定事故    → "60,000 × 40"    = 2,400,000
重大自然灾害       → "60,000 × 40"    = 2,400,000
客运列车/航空意外   → "60,000 × 60"    = 3,600,000
```

#### 安鑫优选终身护理

```
等待期             → "90日"
护理金 18岁前      → "2,400 × 10"                    = 24,000（现金价值不展示）
护理金 18-61岁     → "2,400 × 10 × 160%"             = 38,400
护理金 61岁后      → "max(2,400 × 10 × 120%, 60,312)" = 60,312
```

### 2.4 被保人汇总 buildMemberAnnualSummaries()

```
输入: PolicyCashflowPlan[]
输出: MemberAnnualSummary[]
```

1. 按被保人分组所有 `PolicyCashflowPlan`（排除 `expired = true`）
2. 合并同一被保人所有保单的 `annualEntries`
3. 同一年份的 `amount` 相加得 `totalAmount`
4. 逐年计算 `cumulative`
5. 保留 `details` 数组方便展开查看来源

## 6. 边界情况与容错

### 6.1 已过期保单

保障结束日期 < 当前日期的保单：
- 被保人汇总表不计入（已无未来领取）
- 单保单明细仍展示场景赔付表（历史可查）
- UI 显示灰色"已过期"标签

### 6.2 生日缺失

- 年度现金流表不生成（无法计算年龄轴）
- 场景赔付表正常展示
- 页面顶部 notes 提示，附"编辑保单"链接

### 6.3 生效日缺失

- 年度现金流表不生成
- 场景赔付表不受影响

### 6.4 condition 无法解析

- 该指标不展开为年度条目
- notes 记录提示，不阻断其他指标渲染

### 6.5 现金价值/账户价值缺失

- 硬规则：没有数据就不展示，绝不硬写
- 年度现金流表不显示"现金价值"列（整列隐藏）
- 场景赔付表中涉及现金价值的部分显示"按账户实际价值"

### 6.6 多保单同年重叠

- 汇总表中年金额 = 各保单领取之和
- 点击该年展开查看来源明细
