# 家庭保单报告雷达图 — 设计文档

> 日期: 2026-05-31
> 状态: 待用户确认
> 范围: 家庭保障分析报告中的全家保障均衡雷达、家庭成员保障对比雷达、金额型雷达数据模型、SVG 展示和测试

## 目标

在现有“家庭保障分析报告”中新增两类雷达图:

1. 全家保障均衡雷达: 放在报告头部统计卡下方，用一张图展示全家在重疾、意外、医疗、寿险、财富五个方向的金额结构。
2. 家庭成员保障对比雷达: 放在“被保人保单明细”前，用同一套五维对比不同家庭成员的保障金额差异。

雷达图底层使用真实金额，图形分值只用于本家庭内部相对展示，不表达行业达标率，也不判断保障是否足够。

## 非目标

- 不新增图表库依赖。
- 不新增后端接口。
- 不引入行业标准保额阈值。
- 不把雷达分数包装成专业达标分、健康分或推荐分。
- 不做产品推荐。

## 页面位置

报告区块顺序调整为:

1. 报告头部
   - 标题、生成时间、全家总统计
   - 全家保障均衡雷达
2. 待关注事项
3. 家庭保单清单
4. 家庭成员保障对比雷达
5. 被保人保单明细
6. 重疾分析
7. 意外分析
8. 财富分析

这样总雷达服务于第一眼概览，成员对比雷达承接后面的被保人明细。

## 数据模型

在 `FamilyReport` 中新增 `radar` 字段，由 `src/family-report-engine.mjs` 生成。

```ts
type FamilyRadarReport = {
  dimensions: FamilyRadarDimension[];
  family: FamilyRadarSeries;
  members: FamilyRadarSeries[];
  hiddenMembers: FamilyRadarSeries[];
};

type FamilyRadarDimension = {
  key: 'critical' | 'accident' | 'medical' | 'life' | 'wealth';
  label: string;
};

type FamilyRadarSeries = {
  name: string;
  scores: FamilyRadarScore[];
  totalAmount: number;
  notes: string[];
};

type FamilyRadarScore = {
  key: FamilyRadarDimension['key'];
  label: string;
  amount: number;
  score: number;
  amountText: string;
  policyCount: number;
  note: string;
};
```

`amount` 是真实金额，`score` 是绘图用的 0-100 相对值。

## 金额口径

五个维度统一以可落地金额作为底层值。

### 重疾

- 优先取重疾首次给付金额。
- 多次给付、中症、轻症、特定疾病等作为辅助说明，不直接累加到重疾主金额，避免虚高。
- 如果只有公式但无法落数，不进入 `amount`，在 `note` 中显示“公式型待确认”。

### 意外

- 汇总一般意外、交通意外、航空意外、自驾/驾乘、公共交通等可识别责任金额。
- 同一保单同一场景避免重复堆叠。
- 医疗报销类意外责任优先归入医疗，除非现有责任明确只属于意外医疗额度。

### 医疗

- 从保单类型、产品名、责任文本、coverage indicators 中识别医疗、住院、门诊、报销、百万医疗、手术等责任额度。
- 可落数的医疗额度进入 `amount`。
- 只识别到报销机制但没有额度时，金额为 0，并在说明中提示“额度待确认”。

### 寿险

- 从身故、全残、终身寿、人寿保障、护理险中的身故/全残责任等识别金额。
- 与重疾、意外已经归类的责任不重复计入。

### 财富

- 财富金额 = 最新现金价值 + 未来现金流累计领取。
- 最新现金价值来自每张保单现金价值表中 policyYear 最大且 cashValue 有效的一行。
- 未来现金流累计领取来自 `cashflowEntries.amount` 的合计。
- 展示时必须拆分显示“现金价值”和“未来领取”，不能只显示合计。

## 相对分计算

### 全家保障均衡雷达

先计算全家五个维度的真实金额。五个维度中最高金额映射为 100，其它维度按比例换算:

```txt
familyScore = dimensionAmount / maxFamilyDimensionAmount * 100
```

如果五个维度金额全部为 0，则不绘制多边形，展示空态。

### 家庭成员保障对比雷达

成员对比按维度单独归一化。每个维度中，成员最高金额映射为 100，其它成员按比例换算:

```txt
memberScore = memberDimensionAmount / maxMemberAmountForSameDimension * 100
```

如果某个维度所有成员金额都为 0，则该维度所有成员分数为 0。

这意味着成员对比图表达“这个家庭内部谁在某个维度更强或更弱”，不表达行业达标。

## 成员展示规则

- 成员对比雷达最多叠加 4 个成员。
- 默认展示用于对比价值最高的成员:
  - 先按 `totalAmount` 从高到低取前 3 名。
  - 再补入 `totalAmount` 最低的 1 名。
  - 去重后如果不足 4 名，再按 `totalAmount` 从高到低补齐。
  - 这样同时展示高保障成员和低保障成员，避免只展示强者。
- 超过 4 个成员时，其余成员放入 `hiddenMembers`，在雷达图旁或下方用文字列出短板摘要。
- 图例必须显示成员姓名和颜色。

## UI 设计

新增可复用 SVG 组件:

- `RadarChart`
  - 输入 dimensions 和 series。
  - 输出 SVG 网格、轴标签、数据多边形、图例。
  - 提供 `aria-label`。
- `FamilyRadarSection`
  - 展示全家保障均衡雷达。
  - 显示五维真实金额摘要。
  - 财富维度拆分显示现金价值和未来领取。
- `MemberRadarSection`
  - 展示家庭成员保障对比雷达。
  - 显示成员图例。
  - 显示成员短板摘要和未展示成员摘要。

移动端采用单列布局，雷达图宽度自适应。导出图片时保留 SVG，不需要 canvas 重绘。

固定说明文案:

```txt
雷达图按本家庭内部金额比例绘制，非行业达标分。
```

## 错误与缺失数据

- 某维度没有金额: `amount` 为 0，`amountText` 显示 `0元`，`note` 显示“未识别到可落地金额”。
- 公式型金额无法落数: 不进入雷达金额，显示“公式型待确认”。
- 财富缺现金价值表: 财富只计算已有未来现金流，提示“现金价值待上传/待识别”。
- 财富缺现金流: 财富只计算最新现金价值，提示“未来领取待识别”。
- 五个维度金额全部为 0: 展示空态，不绘制误导性多边形。
- 成员超过 4 个: 图上只展示 4 个，其余成员以文字摘要保留。

## 测试要求

### 引擎测试

在 `tests/family-report-engine.test.mjs` 覆盖:

- `FamilyReport` 包含 `radar` 字段。
- 五维金额聚合正确。
- 全家雷达按五个维度最大金额归一化。
- 成员雷达按同一维度的成员最大金额归一化。
- 财富金额等于最新现金价值加未来现金流累计领取。
- 公式型或缺失金额不进入雷达底层金额。
- 成员超过 4 个时，`members` 和 `hiddenMembers` 分离。

### UI 测试

在 `tests/customer-ui-style.test.mjs` 覆盖:

- 页面包含“全家保障均衡雷达”。
- 页面包含“家庭成员保障对比雷达”。
- 全家雷达位于家庭保单清单之前。
- 成员对比雷达位于被保人保单明细之前。
- SVG 包含可访问标签。
- `package.json` 不新增图表库依赖。

## 实施边界

第一版只基于现有本地保单、责任、现金价值和现金流数据生成雷达图。后续如果要做行业达标分、收入负债驱动的建议保额、产品推荐或专业风险评分，需要单独设计，不放进本次实现。
